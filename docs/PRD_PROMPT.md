## 프로젝트 컨텍스트

**프로젝트명:** 고전시가 웹툰 쇼츠 자동 생성 파이프라인

**한 줄 설명:**
평가원 고전시가 PDF를 입력받아, AI가 대본을 작성하고 웹툰 이미지를 생성한 뒤
TTS 음성을 합성하여 유튜브/틱톡/인스타릴스용 쇼츠를 자동으로 만들고 업로드합니다.

**기술 스택:**
- Python 3.11
- ComfyUI (로컬 API, port 8188) — Anything V5 체크포인트 사용
- TTS: Edge-TTS (무료, 로컬) / ElevenLabs (유료, API)
- 영상 합성: FFmpeg + MoviePy
- LLM: Claude API (대본 생성)
- PDF 파싱: pdfplumber 또는 pymupdf
- 업로드: YouTube Data API v3, (선택) TikTok / Instagram Graph API

**전체 파이프라인 (순서):**
1. PDF 파싱 → 고전시가 텍스트 추출
2. LLM 대본 생성 → 장면별 내레이션 + 이미지 프롬프트 분리
3. ComfyUI 이미지 생성 → 장면별 웹툰 컷 저장
4. TTS 음성 합성 → 장면별 MP3/WAV 생성
5. FFmpeg/MoviePy 영상 합성 → 이미지 + 음성 → MP4 쇼츠
6. 플랫폼 업로드 → 유튜브(필수), 틱톡/인스타(선택)

**MVP 범위:**
- 단일 고전시가 1편 처리
- 유튜브 업로드까지 end-to-end 자동화
- CLI 실행 방식 (웹 UI 없음)
- 중간 결과물은 모두 디스크 캐시
- 재실행 시 이미 생성된 단계는 스킵

**MVP 제외 범위:**
- 웹 대시보드 / 관리 UI
- 배치 처리 (여러 시가 동시 처리)
- 틱톡 / 인스타 자동 업로드
- 음악 배경 삽입
- 자막 번역 (현대어 번역은 대본에 포함)

## PRD 작성 지침

다음 섹션을 포함하여 PRD를 작성하세요.
각 섹션마다 **구체적인 수치와 기준**을 반드시 명시하세요. "적절히", "빠르게" 같은 모호한 표현은 금지입니다.

### 필수 섹션 목록

1. **개요 (Overview)**
   - 프로젝트 배경 및 존재 이유 (Why now?)
   - 핵심 가치 제안 1~3줄

2. **목표 및 성공 지표 (Goals & Success Metrics)**
   - MVP 목표 (정성적)
   - 측정 가능한 KPI 3~5개 (예: "1편 처리 시간 ≤ 10분")

3. **사용자 및 이해관계자 (Users & Stakeholders)**
   - 1차 사용자 페르소나
   - 이해관계자 목록

4. **기능 요구사항 (Functional Requirements)**
   - 각 파이프라인 단계를 Feature로 분리
   - 각 Feature에 대해: 설명 / 입력 / 출력 / 우선순위(P0/P1/P2) / 완료 기준(Acceptance Criteria)
   - 표 형식 권장

5. **비기능 요구사항 (Non-Functional Requirements)**
   - 성능: 단계별 최대 처리 시간
   - 안정성: retry 정책, 실패 시 동작
   - 확장성: MVP 이후 배치 처리 고려 여부
   - 보안: API 키 관리 방식
   - 캐시: 중간 산출물 저장 경로 및 무효화 조건

6. **기술 아키텍처 개요 (Technical Architecture Overview)**
   - 파이프라인 다이어그램 (Mermaid flowchart로 작성)
   - 모듈별 책임 한 줄 설명
   - 외부 의존성 목록 (서비스명 / 용도 / 필수여부)

7. **데이터 흐름 (Data Flow)**
   - 각 단계의 입력 파일 형식, 출력 파일 형식, 저장 경로 규칙
   - 예시 디렉토리 구조 포함

8. **리스크 및 가정 (Risks & Assumptions)**
   - 기술적 리스크 (ComfyUI 응답 지연, API 비용 초과 등)
   - 비즈니스 리스크 (저작권, 플랫폼 정책 등)
   - 각 리스크에 완화 방안(Mitigation) 명시

9. **마일스톤 및 일정 (Milestones & Timeline)**
   - Phase 0 ~ Phase N으로 단계 분리
   - 각 Phase의 목표, 완료 기준, 예상 기간

10. **미결 사항 및 향후 논의 (Open Questions)**
    - MVP에서 결정하지 않은 사항
    - 다음 버전에서 다룰 기능 백로그

## 이 메타 프롬프트 사용법

1. 위 코드블록 안의 텍스트를 **전체 복사**합니다.
2. Claude 대화창에 붙여넣고 전송합니다.
3. Claude가 생성한 PRD를 `docs/PRD.md` 로 저장합니다.
4. 내용 검토 후 수정이 필요한 섹션만 "섹션명 재작성해줘" 로 요청합니다.

## 추가 커스터마이징 팁
