## Part D: 데이터 전처리 파이프라인

```
원본 이미지 수집
    ↓
1. 해상도 필터링 (512×512 이상만)
2. 품질 필터링 (CLIP Score 기반 저품질 제거)
3. 중복 제거 (perceptual hash)
4. 크롭/리사이즈 (512×912, 9:16 비율)
5. 캡셔닝 (BLIP-2 또는 LLaVA로 자동 캡션 생성)
6. 메타데이터 JSON 생성
    ↓
학습 준비 완료 (이미지 + 캡션 쌍)
```

### 캡션 형식 (LoRA 학습용)
```
korean webtoon style, [씬 내용], [캐릭터 묘사], [배경], [감정]
```

## Part E: LoRA 학습 계획

### 학습 방식: Dreambooth LoRA (FLUX.1 기반)

**도구:** kohya_ss (Windows 지원, GUI) 또는 SimpleTuner (FLUX.1 공식 권장)

**설정값 (RTX 4070 Laptop 기준):**
```
rank: 16~32
batch_size: 1 (VRAM 8GB 한계)
learning_rate: 1e-4
steps: 1000~3000
학습 시간 예상: 2~4시간
```

### 학습 단계
1. **1차 LoRA:** 웹툰/만화 스타일 전반 (500~1000장)
2. **2차 LoRA:** 조선시대 풍속화 스타일 특화 (200~500장)
3. **캐릭터 LoRA:** 특정 주인공 캐릭터 일관성 (30~50장, Dreambooth)

## Part F: 캐릭터 일관성 및 컷 연결성 전략

### 캐릭터 일관성
- Dreambooth로 주인공 캐릭터 LoRA 별도 학습
- 모든 씬에 캐릭터 LoRA를 고정 강도(0.8)로 적용
- 프롬프트에 캐릭터 트리거 워드 필수 포함

### 컷 연결성 (7개 → 20~30개로 확장)
- Step 1 NLP에서 씬 분할 기준을 "문장 단위"로 세분화
- 각 씬에 "이전 씬 배경 정보"를 컨텍스트로 전달
- img2img 워크플로우: 이전 씬을 레퍼런스로 사용 (ComfyUI IPAdapter)

### IPAdapter 활용 (연속성 확보)
```
씬 N 이미지 → IPAdapter reference
    + 씬 N+1 프롬프트
    → 자연스럽게 연결된 씬 N+1 이미지
```

## Part G: 지속적 개선 학습 (Continual Learning)

```
생성 → QC 검수 → 불합격 이미지 분석
    ↓
문제 패턴 기록 (comfyui_lessons.md)
    ↓
추가 학습 데이터 수집 (문제 유형에 맞는 이미지)
    ↓
LoRA 재학습 또는 파인튜닝
    ↓
QC 통과율 측정 → 개선 확인
    ↓
반복
```

### 메트릭 추적
- QC 합격률 (목표: 80% 이상)
- 프롬프트-이미지 CLIP Score (목표: 0.28 이상)
- 캐릭터 일관성 점수 (얼굴 유사도, 씬간 비교)
