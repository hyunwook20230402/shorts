# 코드 스타일 규칙

- 들여쓰기 : 스페이스 2칸
- 변수명 : snake_case 사용 (세부 규칙은 `naming_conventions.md` 참조)
- 함수명 : 동사로 시작 (예 : `extract_text`, `generate_audio`)
- 한글 주석 사용
- **타입 힌팅 (Type Hinting):** Python 3.10+ 문법을 활용하여 모든 함수의 매개변수와 반환값에 타입 힌트를 필수로 명시하세요.
- **단일 책임 원칙 (SRP):** 하나의 함수는 오직 한 가지 역할만 수행하도록 모듈화하세요.
- **디스크 캐싱:** 파이프라인의 각 단계(Step 0~5)에서 생성된 중간 결과물(이미지, 오디오 등)은 재사용 및 디버깅을 위해 로컬 디스크에 임시 저장(캐싱)하는 로직을 포함하세요.
- **캐시 경로 함수:** 각 Step 모듈(`stepN_*.py`)에 `get_cache_path()` 함수를 정의하고, `pipeline_runner.py`는 반드시 이 함수를 import하여 사용하세요. 직접 경로 조합 금지.
- **ComfyUI 응답 파싱:** `.get()` 방어 코딩 사용 (KeyError 방지)
- **load_dotenv() 호출:** 모듈 최상단에서 1회만 호출
