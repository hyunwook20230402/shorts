## 현상

이미지에 텍스트 구역이 상단/하단으로 분리된 경우, HCX-005가 상단 4줄만 추출하고 하단 4줄을 누락하는 패턴이 확인됨.

- **발생 케이스**: `test_data/test1.png` (2026-04-05)
- **이미지 구조**: 상단 4줄(현대어 번역 형태) + 하단 4줄(원문) + EBS 워터마크
- **수정 전 결과**: 4줄만 추출 (하단 구역 전부 누락)
- **수정 후 결과**: 8줄 완전 추출 성공

## 근본 원인 (추정)

1. **현대어 오판 제외**: 상단 텍스트가 현대어처럼 보이면 OCR 모델이 "원문 아님"으로 판단하여 이후 구역 탐색을 중단할 가능성
2. **단일 구역 가정**: 기본 OCR 프롬프트가 하나의 텍스트 블록만 처리한다고 암묵적으로 가정
3. **하단 구역 탐색 중단**: 첫 번째 구역 추출 완료 후 스캔을 멈추는 동작

## 해결책 (적용된 프롬프트 수정)

1. **Rule 1 완화**: "해석/풀이/설명 금지" → "이미지 안에 인쇄된 텍스트는 모두 원문으로 간주하여 추출"
   - 현대어처럼 보여도 원문으로 간주하도록 강제
2. **Rule 11 추가**: 이미지에 텍스트 구역이 여러 개 있으면 위→아래 순서로 모두 연결 추출
3. **User 프롬프트 강화**: 상단·하단 복수 구역 추출 지시 명시

## 검증 결과 (2026-04-05)

- 이미지 추정 줄 수: 8줄
- OCR 추출 줄 수: 8줄
- 첫 줄 일치: "비 맞으며 논바닥에 엎드려 김매니" ✓
- 마지막 줄 일치: "어찌 이리도 극성스레 침탈하는가" ✓
- EBS 워터마크 오인식: 없음 ✓
- 판정: 합격

**Why:** HCX-005가 복수 구역 이미지에서 첫 구역만 추출하는 경향이 있음. 명시적인 위→아래 전체 추출 지시가 필수.

**How to apply:** 복수 구역 이미지(상단/하단 분리, 좌우 분리 등)를 처리할 경우 항상 Rule 11 계열의 전체 구역 추출 지시를 OCR 프롬프트에 포함할 것.
