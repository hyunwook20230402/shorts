## 현상

고전시가 원문이 현대어처럼 읽히는 경우(현대국어 어휘, 자연스러운 문장 구조), HCX-005 OCR이 해당 텍스트를 "해석/번역문"으로 오인하여 추출 대상에서 제외함.

- **발생 케이스**: `test_data/test1.png` 상단 4줄 (2026-04-05)
- **해당 텍스트**: "비 맞으며 논바닥에 엎드려 김매니", "흙투성이 험한 꼴이 어찌 사람 모습이랴만" 등
  - 형태는 현대어에 가깝지만 이미지에 인쇄된 실제 원문 텍스트
- **수정 전 동작**: 이 4줄이 제외되어 하단 4줄만 추출되었을 가능성 (또는 반대)
- **실제 패턴**: 상단 4줄은 추출, 하단 4줄 누락 → 복수 구역 문제와 복합 발생

## 근본 원인

OCR 프롬프트 Rule 1이 "해석/풀이/설명 금지"로 명시되어 있을 때, 모델이 현대어 형태의 원문을 "번역/해석"으로 오판할 수 있음.

## 해결책

**프롬프트 규칙**: "이미지 안에 인쇄된 텍스트는 현대어든 고어든 구별 없이 모두 원문으로 간주하여 추출"
- OCR 단계에서는 텍스트의 시대적 형태를 판단하지 않도록 명시
- 판단은 Step 1 NLP에서 수행

## 고전시가 OCR 검증 시 주의사항

- 이 패턴 때문에 "고어 어미 미탐지 = 경고"가 항상 적절하지 않음
- 현대어 번역체 원문 이미지의 경우 고어 어미가 없는 것이 정상
- OCR validation에서 고어 어미 체크는 info 수준으로만 처리할 것 (warning/error 아님)

**Why:** 고전시가 이미지에는 원문과 번역문이 혼재하거나, 현대국어 표기 원문이 포함될 수 있음. 형태만 보고 원문 여부를 판단하면 정당한 추출 대상을 제외하게 됨.

**How to apply:** OCR 프롬프트에서 "이미지에 인쇄된 텍스트 = 원문"이라는 원칙을 명시. 번역문 여부 판단은 OCR 단계가 아닌 NLP 단계에서 수행.
